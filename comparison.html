03 Fees & Pricing Model

Trusted Agents

Custom

Consulting engagements ("Expert Burst", GTM sprints). Priced per scope — no public rate card.

Visa Trusted Agent Protocol

Free

Open protocol on GitHub & Visa Developer. No license fee. Standard Visa scheme fees still apply on settled transactions.

Stripe Agentic Commerce Suite

Stripe rates

Standard Stripe processing (~2.9% + 30¢, region-varies). No new line item for agentic — uses existing Checkout / Tax / Radar pricing.

Stripe Projects

Pass-through

Free CLI; you pay providers (DB / auth / analytics) at their rates via Shared Payment Token. Stripe takes no extra cut to provision.

Mastercard Agent Pay

Network fees

Issuer-priced. No public agentic surcharge announced — runs on existing Mastercard interchange + tokenization infrastructure.

04 How We Can Use Each — Possibilities for Our Stack

Trusted Agents — as a playbook

  • Borrow their delegation / consent UX patterns for our agent flows.
  • Use their framing for risk reviews with legal & compliance.
  • Reference their GTM playbook when launching agent-facing SKUs.

Visa TAP — at our merchant edge

  • Adopt the open spec on our checkout / API edge so trusted bots aren't blocked by WAFs.
  • Verify signatures to grant agents lower friction (no CAPTCHA, no rate-limit) while bouncing scrapers.
  • Pair with our existing analytics to measure agent-driven traffic separately.

Stripe Agents — sell through agents

  • Push our catalog as a product feed to ChatGPT / Claude / Perplexity surfaces.
  • Use Checkout Sessions + SPT so the agent completes purchase without leaving the chat.
  • Receive order events into the same fulfillment pipeline we use today.

Stripe Projects — agent-built infra

  • Let our internal Claude / Cowork agents spin up DBs and auth without us pasting API keys.
  • Centralize provider credentials in the Stripe vault — fewer .env files leaking.
  • Use SPT-based upgrades so the agent can move from free → paid tier on its own budget.

Mastercard Agent Pay — issuer-grade payments

  • If we issue cards (e.g., corporate / virtual), enroll them as Agentic Tokens for B2B procurement bots.
  • Pass user-intent context with each transaction → better issuer approval rates on agent purchases.
  • Use the consent layer to set per-agent spend caps & merchant allowlists.

Combined stack — recommended architecture

  • Edge: Visa TAP to verify the agent.
  • Commerce: Stripe Agents for catalog + checkout.
  • Payment rail: Mastercard Agent Pay tokens where the issuer supports them.
  • Internal infra: Stripe Projects for our own agent-driven ops.
  • Strategy: Trusted Agents patterns for governance.

05 Verdict — Quick Reads

Pick Visa TAP if…

You want a free, open, standards-based way to tell good bots from bad on your own site — the cheapest insurance against being either scraped or accidentally blocking real customers.

Pick Stripe Agents if…

You're already on Stripe and want the fastest path to selling inside ChatGPT-style agents without rebuilding checkout, tax, and fraud.

Pick Stripe Projects if…

Your developers (or AI agents) keep needing to spin up infra — and you'd rather pay providers via a vaulted token than juggle 12 billing portals.

Pick Mastercard Agent Pay if…

You're a bank, fintech, or large enterprise and need issuer-side controls + risk scoring on AI-initiated payments — not just merchant-side ones.

Pick Trusted Agents if…

You need senior strategy partners to help frame the agentic roadmap — not a piece of code, but a way to think about delegation, trust, and rollout.

Note: This is a structured comparison built from public documentation as of April 2026. Pricing on processing networks (Stripe, Mastercard) varies by region, MCC, and contract — confirm with your account manager before signing off.
